## master公式
**所有子问题规模相同的递归**才能用master公式
T(n) = a \* T(n / b) + O(n ^ c);
a, b, c都是常数

是在一层中：
1) n 为数据量  刚刚的max函数，L-R这里有n个数
2) a 为次数 刚刚的max函数，先找lmax 再找rmax两次调用
3) b为子规模 刚刚的max函数，先均分两份，先找左范围的，所以b就是2
4) O(n ^ c)是除去函数调用的其他行为的时间复杂度
注意，必须子规模相等，比如如果我不均分两份了，2/3份，但是是两个规模都是2/3，那么还是满足的； 但是如果一个是2/3 另一个是1/3那就是不满足的
	必须严格遵守公式，像刚才的2/3的例子
		2 \* T(n \* (2 / 3)) + O(1)
		这里 a = 2  b = 3 / 2(因为公式是除法)  c = 0(n ^ 0 == 1

计算：
1) 如果log(b, a) < c, 复杂度为：O(n ^ c)
2) 如果log(b, a) > c, 复杂度为：O(n ^ log(b, a))
3) 如果log(b, a) == c, 复杂度为：O(n ^ c * logn)

特殊的：
T(n) = 2 \* T(n / 2) + O(n \* logn);时间复杂度为：O(n \* ((logn) ^ 2))
看到不太符合master公式，但是直接记住就行，他没办法用master公式描述，因为master要严格遵循O(n ^ c);
